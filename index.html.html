SuperiorCoin software Hard Fork

Esta última presentación del sistema presenta un numero considerable de cambios que mejoran el funcionamiento de SuperiorCoin, así mismo un gran conjunto de errores corregidos.

SuperiorCoin software Hard Fork actualizaciones de red a partir de Abril 10 comenzó incrementando el tamaño de los anillos mínimos, ordenando las entradas para no perder la billetera por inferencia, cambios ligeros en las pruebas de trabajo para prevenir ataque DoS con ASICs basado en cambios sobre un bloque dinámico de recompensa.

Note esta es una actualización de la red consensual programado, Por lo tanto a diferencia del BCH/BTC no se creara una nueva moneda.

Algunas de las mayores mejoras lanzadas son:

  • Recompensas basadas en el tiempo del bloque dinámico.
  • Cambio PoW para el bloqueo de ataques DoS con ASICs.
  • Adición de entradas ordenada para prevenir inferencias en la cartera.
  • Incremento del tamaño de anillos en 5 a 7.
  • Soporte en Sub-direcciones añadidas.
  • Se agrego soporte multigrado.
  • Soporte para SunOS/Solaris.
  • Inicio de soporte para equipos con cartera Ledger Nano S.
  • Adición de semilla con contraseña encriptada.
  • Forma segura en eliminación de llaves en memoria, para mayoría de los casos inutilizables.
  • Inicio de la implementación a prueba de fuego en vivo de testnet.
  • Se agregaron cuenta, etiquetas y rótulos atreves de sub-direcciones.
  • Se agrego un soporte rustico para OMQ.
  • Implementación sobre medidas de mitigación para Forks en llaves que amenazan la privacidad.
  • Nueva red que refleja las características de las redes principales.
  • Adición de soporte SSL para carteras ligeras API.
  • Adición de soporte CORS para los stack del RPC.
  • Adición de bandera para la billetera cliente(CLI) --generate-from-spend-key.
  • Adición de bandera para el demonio –disable-dns-checkpoints.
  • Mejoras masivas para construir el endurecimiento.
  • Se agregaron pruebas (fuzz) nativas para la entrada de usuarios.
  • Se agrego la habilidad de limitar conexiones entrantes.
  • ¨fluffy blocks¨ Habilitados por defecto.
  • Adición del espacio limitado en bandera mempool –max-txpool-size.
  • Aceptación de nodos remotos para el proceso temporal de la RPC durante la solicitud IBD.
  • Se agrego un comando relay_tx a la billetera RPC.
  • Aceptación para la recuperación de la llave de gasto atreves de una llamada de la billetera RPC.
  • Las bibliotecas creadas usan código de posición independiente.
  • Mejoras echas en la Imagen Docker.
  • Se agrego un comando sweep_single a la billetera CLI.
  • Los códigos de errores en RPC son mas específicos.
  • Se mejoro la impresión del rastreo de pila con observación en librería.
  • Se dividió y refactorizo wallet_api del código en billetera.
  • Se agrego prioridad en argumentos para sweep_all y donaciones en comandos de la billetera CLI(cliente).
  • Se agrego una opción –do-not-relay para la billetera CLI(cliente).
  • Pruebas de unidad ¨hashchain¨ funcionan nuevamente.
  • Se agrego una prueba para la expansión de sub-direcciones.
  • Se agrego una prueba de rendimiento RingCT.
  • Se agregaron instrucciones en paquetes de instalación para Void Linux.
  • Se crearon cambios para garantizar que no se registren datos confidenciales accidentalmente.
  • Se agrego lista de palabras mnemonic Croata.
  • Se agrego lista de palabras mnemonic Lojban.
  • Se agrego traducción sueca CLI.
  • Se agrego un tiempo limite para conexiones que no completan el ¨handshake¨.
  • Construcción de prueba movido hasta el final del proceso CMake.
  • Como siempre, gran lista de errores solucionados y mejoras de rendimiento.
